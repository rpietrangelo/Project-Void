IMPORTANT ARCHITECTURE NOTES
-----------------------------
- Service Locator pattern: all services register on Awake, GameManager waits for them
- EventBus: all cross-system communication goes through EventBus.cs events defined in GameEvents.cs
- Economy is server-authoritative: client calls CloudScript, server validates and deducts
- Firebase UID is the link between Firebase Auth and PlayFab (LoginWithFirebaseAsync)
- Never trust client for: resources, timer completion, level validation
- DOTween Pro for ALL UI animation (no Unity Animator for UI)
- TextMeshPro for ALL text (no legacy UI Text)
- UniTask for ALL async (no coroutines for network)
- Addressables for ALL asset loading (no Resources.Load)
- New Input System only (no Input.GetKey)
- Commit format: [PHASE-X][TASK-Y] description

KNOWN ISSUES / WATCH OUT
-------------------------
- FirebaseService and PlayFabService must both be present as components in Bootstrap scene
  before GameManager.Start() runs (it waits for both in ServiceLocator)
- LocalCityState.cs has a dual-mode: local-only (Phase 2) vs backend-connected (Phase 3).
  Phase 3 wires EconomyService in. Make sure to connect EconomyService reference in Inspector.
- Building slot state in Phase 3 comes from PlayFab InternalData["buildings"] (JSON blob)
- Firestore worldTiles write is blocked client-side (security rules); all tile ownership
  changes must go through CloudScript
